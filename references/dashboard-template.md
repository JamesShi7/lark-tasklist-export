# Dashboard 模板说明

## 文件结构

Dashboard 是一个**自包含的 HTML 文件**，所有 CSS、JS、数据都内联在单个文件中，无需服务器即可在浏览器中打开。

## 核心原则：完全动态

**不要假设任何固定的自定义字段名**。不同用户的清单可能有完全不同的字段（有的只有 1 个"优先级"，有的有 5 个不同字段）。所有字段名、筛选器、看板分列、badge 都必须根据实际数据动态生成。

## 需要动态生成的部分

### 1. 标题和元信息

```html
<div class="brand">Task Command Center</div>
<h1 class="title">{清单名称} <span>未完成任务</span></h1>
```

### 2. TASKS 数据数组

每个任务对象用动态 key 存储自定义字段值：

```js
{
  "summary": "任务标题",
  "fields": {                    // 动态字段，key 为字段名
    "{字段名1}": "选项值A",       // 第一个自定义字段，用于看板分列
    "{字段名2}": "选项值X",       // 第二个自定义字段，用于分组视图
    "{字段名3}": "选项值M"        // 更多字段作为 badge 展示
  },
  "due": "今天",
  "due_sort": 0,                 // 0=今天, 1=明天, 2=未来, 3=无截止
  "subtask": 4,
  "assignees": ["Alice", "Bob"],
  "subtasks": [
    {"s": "子任务标题", "done": false, "a": ["负责人"]}
  ]
}
```

### 3. FIELD_ORDER 和 FIELD_COLORS

根据实际自定义字段动态生成。第一个字段用于看板分列，其余字段用于筛选器。

为第一个字段的每个选项分配颜色（按重要性从高到低）：

```
#ff4757 (红) → #ff8c42 (橙) → #4facfe (蓝) → #c8a86e (金) → #6a6a80 (灰蓝) → #3a3a50 (暗灰)
```

如果选项超过 6 个，循环使用。

### 4. 筛选器

动态生成下拉筛选器，数量和名称与实际自定义字段对应。每个字段一个筛选器，ID 使用字段名。

### 5. 统计卡片

根据实际数据动态生成，常见统计项：
- 今天到期
- 明天到期
- 总任务数
- 子任务数
- 第一个字段前两个选项的数量（如"高优先级"）

### 6. 表格视图表头

动态生成列：序号、任务标题、各自定义字段列（名称动态）、截止时间、子任务、负责人

### 7. 看板视图分列

按第一个自定义字段的选项值分列。

### 8. 分组视图

按第二个自定义字段（如有）的选项值分组，每组用 emoji 图标（未知选项用 📌）。

## 不需要改动的部分

以下部分从模板直接复制，不需要改动：
- 整体 HTML 骨架
- CSS 样式（CSS 变量体系、暗色主题、动画）
- 三种视图的渲染逻辑（看板/分组/表格）
- 搜索逻辑
- 子任务展开逻辑
- 响应式布局

## 生成方法

1. 读取 `dashboard-template.html` 获取完整的 HTML 骨架和样式
2. 根据已提取的任务数据和字段映射，动态替换所有数据部分
3. 筛选器的数量和名称根据实际字段动态生成
4. 更新页面标题
5. 写入到用户指定目录
