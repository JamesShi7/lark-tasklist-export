# Markdown 输出模板

以下模板中的 `{}` 占位符在实际生成时替换为真实数据。

---

```markdown
# {清单名称} — 未完成任务

> 导出时间：{YYYY-MM-DD HH:mm}
> 数据来源：飞书任务清单 API（实时拉取）
> 共 {N} 条未完成任务

## 统计概览

| 指标 | 数值 |
|------|------|
| 今天到期 | {X} |
| 明天到期 | {X} |
| 有子任务 | {X} |
| 总计 | {N} |

## 任务列表

| # | 任务标题 | {自定义字段1名} | {自定义字段2名} | ... | 截止时间 | 子任务 | 负责人 |
|---|---------|----------------|----------------|-----|---------|--------|--------|
| 1 | {标题} | {字段1值} | {字段2值} | ... | {日期} | {数量} | {姓名1, 姓名2} |
| ... | ... | ... | ... | ... | ... | ... | ... |

{如有用户提供的业务规则，在此注明，例如：}

> 业务规则：{用户描述的业务规则，如字段适用范围等}

## 子任务详情

## 团队成员

| open_id | 姓名 |

```

---

## 排序规则

任务按以下顺序排列：
1. 今天到期（按第一个自定义字段从高到低）
2. 明天到期（按第一个自定义字段从高到低）
3. 未来日期（按日期从近到远）
4. 无截止时间（按第一个自定义字段从高到低）

## 日期格式

- 当天 → "今天"
- 次日 → "明天"
- 带具体时间（is_all_day=false）→ "今天 21:00"、"明天 14:30"
- 其他 → "MM月DD日"

## 关键原则

- **自定义字段列名完全动态**：根据实际推断的字段名生成列头，不同清单可能 0 个、1 个、3 个甚至更多自定义字段
- 不要假设任何固定字段名（如"优先级"、"Project"、"业务组"），一切以 API 返回的 `custom_fields[].name` 为准
